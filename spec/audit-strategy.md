# 審計機制規範 v1.0

最後更新：2026-03-30

## 三層審計架構

本系統採用三層審計機制，確保報告品質與引用準確性。

### Phase 1.5: URL 審計
- 驗證所有來源 URL 的有效性
- 檢查 404 錯誤或內容不符
- 輸出：`audit-urls.md`

### Phase 2.5: 引用比對審計
- 檢查主張與來源的對應關係
- 驗證引用內容的準確性
- 輸出：`audit-claims.md`

### Phase 3.5: 最終審計
- 品質分級（High/Med/Low）
- 數字一致性掃描
- 輸出：`audit-final.md`

---

## 審計標準

| 審計層級 | 檢查項目 | 通過標準 |
|---|---|---|
| URL 審計 | 來源可訪問性 | >90% URL 有效 |
| 引用審計 | 主張準確性 | 無重大矛盾 |
| 最終審計 | 整體品質 | 達到 Medium 以上 |
