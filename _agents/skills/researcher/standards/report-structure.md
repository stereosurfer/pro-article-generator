# Report Structure Template v2.0 / 報告章節架構模板
# Universal standard — category-independent / 通用標準 — 與品類無關

> Last updated / 最後更新: 2026-03-31
> Derived from / 來源: spec/report-template.md v1.0

All chapters are mandatory. If a section lacks sufficient data, keep the heading and explain the gap. Never omit a chapter.
所有章節均為必填。若某小節因資料不足無法完成，保留標題並說明缺口原因，不得省略。

---

## Chapter 0: Executive Summary / 執行摘要

### 0.1 Thesis Statement / 核心論點
Define the product's historical position and technical authority within its domain.
說明該產品在其領域中的歷史方位與技術權威性。

### 0.2 Key Findings Matrix / 關鍵發現矩陣
Structured table presenting:
- Key technical spec summary / 關鍵技術規格摘要
- Major version differences / 主要版本差異摘要
- Cross-cultural market recognition / 跨文化市場認可度
- Overall report credibility rating (based on Chapter 8) / 整體可信度評級

---

## Chapter 1: Historical Lineage & Design Philosophy / 歷史背景與設計哲學

### 1.1 Technological Generation Shift / 技術代際轉型
Position the product in its technological evolution timeline. List predecessors, contemporaries, and successors.
說明該產品在技術演進脈絡中的位置。

### 1.2 Design Intent & Compromises / 設計初衷與妥協
Analyze the industrial standards at time of development and resulting architectural compromises.
分析研發當時的工業標準與設計妥協。

---

## Chapter 2: Technical Architecture & Component Analysis / 技術架構與組件深度解析

### 2.1 Core Transduction / Conversion Principle / 核心換能/轉換原理
Explain the fundamental operating principle and key physical components.
說明核心工作原理與關鍵物理組件。

### 2.2 Circuit Topology & Component Evolution / 電路拓撲與元件演進
Analyze circuit architecture, key active and passive components, and how component changes affected performance.
分析電路架構、關鍵元件及其演進對性能的影響。

### 2.3 Critical Components Deep Dive / 關鍵元件深入分析
Focus analysis on the single most defining component (e.g., transformer, capsule, DAC chip, driver).
針對最具定義性的單一元件進行重點分析。

---

## Chapter 3: Evolution & Revision Analysis / 版本演進與關鍵差異分析

### 3.1 Version Timeline / 版本斷代史
Chronological listing of all known versions with production dates, serial ranges, and manufacturing locations.
以時間軸列出所有已知版本。

### 3.2 Technical-Subjective Correlation / 技術特徵與主觀關聯
Link specific technical modifications to measurable data and subjective descriptions. Clearly mark which correlations have measurement support vs. community consensus.
說明技術修改如何對應量測數據與主觀描述，標明哪些有量測支持。

---

## Chapter 4: Cultural Aesthetics & Regional Usage / 跨文化美學與區域應用研究

### 4.1–4.4 Regional Analysis Sections / 區域分析段落
Analyze the product's reception and usage patterns across different cultural markets.
Structure depends on category (defined in category.md).
分析產品在不同文化市場的接受度與使用模式。具體段落結構由品類設定定義。

### 4.5 Source Attribution / 資料來源說明
Declare which language communities provided data for each section and note coverage gaps.
標明各節資料來自哪種語言社群，並標注不足之處。

---

## Chapter 5: Expert Modifications & Boundary Testing / 專家改裝與極限邊界測試

### 5.1 Master-Level Modification Procedures / 大師級改裝程序
Reference published expert documentation. All modification descriptions must cite original document sources.
參考公開的專家技術文件，所有改裝描述必須附原始文件來源。

### 5.2 Failure Modes & Aging Analysis / 失效模式與老化分析
Explore common failure modes after long-term use and their identifiable impact on performance.
探討長期使用後的常見失效模式及其對性能的影響。

---

## Chapter 6: Comparative Competitive Analysis / 競品對抗與非對應比較

### 6.1 Direct Contemporary Competitors / 當代直接競品
Compare 2-3 competitors from different design philosophies. Focus on selection logic.
從技術規格與美學兩個維度對比 2-3 款競品。

### 6.2 Digital Emulation Challenges / 數位模擬的挑戰
Analyze modern software emulation limits for the product's nonlinear characteristics. Cite comparative tests.
分析現代軟體對該硬體非線性特徵的模擬極限。

---

## Chapter 7: Market Value & Longevity / 市場價值與長期持有評估

### 7.1 Serial Numbers & Investment Value / 序列號與投資價值
Analyze secondary market pricing logic by production era, origin, and version. Cite trackable market data.
分析二手市場溢價邏輯與價格區間。

### 7.2 Maintenance Cost & Parts Availability / 維護成本與料件獲取
Assess core component market availability and long-term ownership risk.
評估核心零件的市場可得性與長期持有風險。

---

## Chapter 8: Research Statistics & Quality Assessment / 研究統計與品質檢驗

### 8.1 Source Matrix / 來源矩陣
Table listing all cited sources with: URL, author/institution, year, language, evidence level, corresponding chapter.
表格列出所有引用來源。

### 8.2 Audit Results Summary / 審計結果摘要
- URL existence check: passed N, failed N, replaced N / URL 存在性查核
- Citation content verification: checked N claims, confirmed N, unverifiable N, removed N / 引用內容比對
- Number consistency scan: contradictions found N, resolution notes / 數字一致性掃描

### 8.3 Blind Spot Declaration / 盲點聲明
- Sections lacking independent lab measurement support / 缺乏獨立量測支持的部分
- Sections relying solely on community consensus / 僅依據社群共識的部分
- Regional markets with insufficient language coverage / 語言資料不足的區域
- Overall report credibility self-assessment (High/Medium/Low with reasoning) / 整體可信度自評
